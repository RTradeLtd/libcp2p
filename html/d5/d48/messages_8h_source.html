libcp2p
A libp2p-like protocol with first class support for embedded systems. End goal is a protocol that can run on anything from your fridge, embedded systems, and anything else. Higher level languages bindings will be done via FFI. Heavy WIP

13 /*! @file messages.h
14  * @brief defines message types and tooling for a very minimal RPC framework
15  * @note when sending messages anytime you send a new message, you must first send a
16  * single byte that indicates the size of the messsage we are sending
17  * @note this helps ensure that we can appropriately handle new requests and allocate
18  * enough memory
19  * @details the message types here are intended to be served as a building block for
20  * your own applications
21  * @details at a minimum the types here are setup to establish a secure
22  * communications channel using ECDSA keys and ECDH key agreement
23  */

180 /*!
181  * @brief used to handle receiving data from a TCP socket
182  * @details it is designed to reduce the manual overhead with regards to processing

184  * @details because the first byte of any data stream coming in defines the size of

186  * @details and the remaining data defines the actual cbor encoded data. therefore we

190  * @param thl an instance of a thread_logger, can be NULL to disable logging

192  * @param max_buffer_len specifies the maximum buffer length we are willing to

194  * @return Success: pointer to a chunk of memory containing the received RPC message
195  * @return Failure: NULL pointer
196  * @warning we will allocate slightly more memory than max_buffer_len since we have
