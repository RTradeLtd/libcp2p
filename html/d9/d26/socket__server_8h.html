libcp2p
A libp2p-like protocol with first class support for embedded systems. End goal is a protocol that can run on anything from your fridge, embedded systems, and anything else. Higher level languages bindings will be done via FFI. Heavy WIP

used for a server to send a message to another server

this is a sort of "bi-directional RPC method" whereby a server can send a

request to another server acting as a client, but enabling either us

or the peer to invoke RPC methods. Essentially it is like handle_inbound_rpc

except it is responsible for sending requests to a remote server, and any responses

from the server are ran through handle_inbound_rpc

Returns
Success: 0
Failure: -1
